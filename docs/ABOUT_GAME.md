# 🏜️ ملك الصحراء — King of the Deserts

## 📋 نظرة عامة
لعبة استراتيجية صحراوية (Desert RTS) تعمل على المتصفح بالكامل — جوال + ديسكتوب.
أنت قائد قبيلة في الصحراء، تبدأ ضعيفاً وتبني إمبراطوريتك عبر:
- بناء وتطوير المباني في الواحة
- تدريب الجيش وجمع الأسلحة الأسطورية
- قتل الوحوش في العالم المفتوح (PvE)
- منافسة اللاعبين الآخرين (PvP + Online Multiplayer)
- السيطرة على الواحات والأقاليم

## 🚀 نظام التشغيل

### التشغيل المحلي (تطوير)
```bash
npm install
npm start
# → http://localhost:3000
```

### البيئة (Production) — Coolify / أي Host
```bash

PORT=3000

```

- **MongoDB** (اختياري — اللعبة تشتغل بدونه مع حفظ في الذاكرة)
- **npm** ≥ 9

|-------|--------|
| `npm start` | تشغيل السيرفر (production) |

## 🎮 شرح اللعبة الكامل

### الشاشات الرئيسية (5 أزرار سفلى)

#### 1. 🐪 الترقية (Promotion)
- إدارة مباني الواحة
- كل مبنى: مقاتلة وحش ← بناء ← إنتاج موارد
- 4 مباني لكل قرية، قريتان حالياً
- ترقية المباني تزيد الإنتاج

#### 2. 🏆 المرتبة (Ranking)
- لوحة المتصدرين من كل اللاعبين
- يُحسب من قاعدة البيانات حسب army_power

#### 3. 🗺️ الأراضي (Territories)
- 5 واحات للسيطرة
- كل واحة: اسم، حالة (محررة/تحت السيطرة)، أيقونة
- احتلال الواحة يتطلب جيشاً قوياً

#### 4. ⚔️ حرب (War)
- **🏟️ الساحة (Arena)**: PvP — عالم مفتوح متعدد اللاعبين
- **🧭 المغامرة (Adventure)**: PvE — وحوش الصحراء
- **🎯 الحملة (Campaign)**: PvE — 1500+ مستوى

#### 5. 👑 التحالف (Alliance)
- الانضمام إلى تحالف
- التبرع للتحالف لرفع المستوى
- مكافآت قوة إضافية

### نظام WebSocket — الملتيكاملة الفورية

- **بروتوكول**: WebSocket عبر `/ws/world`
- **تحديثات الموقع**: كل 100ms (حركة سلسة)
- **حفظ في MongoDB**: كل 5 ثوانٍ
- **الأحداث**:
  - `join` — دخول اللاعب
  - `update` — تحديث الموقع والقوة
  - `monster_killed` — قتل وحش
  - `pvp_attack` / `pvp_notify` — هجوم PvP
  - `player_joined` / `player_left` — إشعارات الدخول/الخروج

### نظام العملات (7 عملات)

| العملة | الأيقونة | الاستخدام |
|--------|----------|-----------|
| 💵 المال (Cash) | 💵 | شراء الجنود، ترقية المباني |
| 🪙 الذهب (Gold) | 🪙 | ترقية الأسلحة |
| 💎 الجواهر (Gems) | 💎 | بونص ترحيبي، مكافآت |
| 👑 King Coins | 👑 | مكافآت نادرة |
| 🔨 المطارق (Hammers) | 🔨 | موارد بناء |
| 📜 المخطوطات (Scrolls) | 📜 | أبحاث وترقيات |
| 📯 الأبواق (Horns) | 📯 | استدعاء التحالف |

### نظام المستويات (Leveling)
- اكتساب XP من قتل الوحوش وإكمال المهام
- كل مستوى: يزيد قوة الجيش، قدرة الاستيعاب، الضرر
- 110 مستوى أقصى

### نظام الجيش
- **القائد (الشيخ)**: 120 HP، 140 سرعة، 12 ضرر
- **الجنود**: 8 جنود، 40 HP لكل جندي
- **الأسلحة**: 7 أسلحة (سيف بدوي → سيف الأساطير)
- كل سلاح: 50 مستوى ترقية

### نظام المعركة
- **PvE**: قتل الوحوش في العالم المفتوح → مكافآت 💵 + 🪙
- **PvP**: هجوم على لاعب آخر → مقارنة قوى + مكافآت
- **Wipe**: موت القائد وكل الجيش → خسارة أرباح الجلسة

### نظام الوحوش
- 12 وحشاً مشتركاً بين كل اللاعبين
- أنواع: ذئب صحراوي، محارب ظل، زعيم الرمال
- إعادة إحياء بعد 25 ثانية

### نظام التحالف
- 4 مستويات ترقية
- كل مستوى: ضرر + دفاع + دخل موارد إضافي
- تكلفة الترقية بالذهب

### شجرة الترقيات (4 مسارات × 4 مستويات)
1. **الضرر (Damage)**: 50/120/250/500 🪙
2. **الدفاع (Defense)**: 40/100/200/400 🪙
3. **السعة (Capacity)**: 60/150/300/600 🪙
4. **السرعة (Speed)**: 35/90/180/360 🪙

### الواحات (5 للسيطرة)
| الواحة | الحالة الافتراضية |
|--------|-------------------|
| 🌴 واحة البداية | 🟢 محررة |
| 🌵 واحة النخيل | 🔴 تحت السيطرة |
| 💎 واحة الكنز | 🔴 تحت السيطرة |
| 🏺 واحة الأساطير | 🔴 تحت السيطرة |
| 👑 واحة الفراعنة | 🔴 تحت السيطرة |

## 🔌 API Reference

### REST Endpoints
| الطريقة | المسار | الشرح |
|---------|--------|-------|
| GET | `/health` | صحة السيرفر وقاعدة البيانات |
| GET | `/api/players` | كل اللاعبين (`?all=true`) |
| GET | `/api/players/:username` | لاعب واحد |
| POST | `/api/players/:username` | إنشاء/تحديث لاعب |
| GET | `/api/leaderboard` | ترتيب اللاعبين |

### WebSocket Events
```
→ { type: "join", username, x_position, y_position, army_power, kills, coinsEarned, unitLevel, armyAlive }
← { type: "world_players", list: [...] }
← { type: "world_monsters", list: [...] }
→ { type: "update", x_position, y_position, army_power, kills, coinsEarned, unitLevel, armyAlive }
← { type: "monster_killed", id, killedBy }
→ { type: "monster_killed", id }
→ { type: "pvp_attack", target, myPower }
← { type: "pvp_notify", attacker, power }
← { type: "player_joined", username }
← { type: "player_left", username }
```

## 🧠 المبادئ الأساسية للتطوير

1. **Vanilla JS فقط** — لا React, لا Vue, لا مكتبات خارجية
2. **RTL إلزامي** — الاتجاه من اليمين لليسار
3. **جوال أولاً** — التجربة تبدأ من 375px
4. **Canvas 2D** — كل الرسم داخل اللعبة على Canvas
5. **WebSocket فوري** — حركة سلسة 100ms
6. **MongoDB** — حفظ دائم (مع fallback للذاكرة)
7. **لا تكرار كود** — كل نظام في ملف منفصل
8. **Closures** — كل شيء داخل `DOMContentLoaded`
9. **Pointer Events** — موحدة للمس والماوس
10. **Auto-save** — كل 15 ثانية في localStorage + كل 5 ثوانٍ في MongoDB
